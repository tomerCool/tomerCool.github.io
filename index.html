TheOdd1sOut Tetris Edition

A 2D vertical scroller fan game about the youtuber TheOdd1sOut, made back in 2017 when I was 13. The levels are generated semi-procedurally and get harder as you progress. Though you might see similar parts more then once, it's highly unlikely to get the same level twice. FRC

I've been a part of Ironi Dalet's robotics team since 10th grade, we take part in the First Robotics Competition. I'm a part of the Control and Programming sub-team and were in charge of writing code for the robot. Some of the cooler things we do include:

  • Computer vision and controlling the robot using data from the camera
  • Making the robot move on its own move without input from the driver during the autonomous period (first 15 seconds of each game)
  • Teaching students who are new to the team how to program and how to write code for a robot

Our code is open source and available on github.

Some team members, myself included, mentor students from nearby elementary schools as part of the First Lego League Jr. program. Teaching them basic engineering and programming principles using Legos and block programming.


Programming Courses

We have annual meetings in the robotics team to discuss our community work, we brainstorm on how it can be improved and what more can we do. In one of those meetings I suggested we teach students in our school how to program and offered to do so. I wanted us to teach programming as I knew it's useful information that can benefit anyone, especially young students. My friends loved the idea and offered to help, and so we decided to make and teach the course.

The course was aimed at students from grades 7 to 9 with no previous knowledge in programming and consisted of 5 weekly meetings, 1½ hours each. We chose to teach JavaScript and p5.js as it's simple to understand so it'll be easy to learn without previous experience, but also useful knowledge to have if they decide to continue learning programming after the course. We also taught them some basic HTML because we wanted them to understand how the technologies that they were learning are used in the real world by experienced programmers.

The course's presentations can be found here.


MultiGravity

Made when I was 13, MultiGravity is my first multiplayer game. All players connected to a joined world where they could place blocks and move around, players were able to switch their gravity upside down (as you might have guessed from the name), and I've also implemented a small pixel art editor so players could draw their own avatars.

As I was already familiar with JavaScript I decided to use Node.js for the back-end, I also used express for serving static files, and the WebSocket API using socket.io for transmitting data back and forth. The game's logic was handled on the server and the client only needed to do the rendering and to send and receive input and rendering instructions.

The back-end's design was quite flawed, I managed to drastically slow it down by duplicating its tab over and over, connecting too many clients at the same time. I was also able for example to change my avatar image more then once by sending a request to the server with Base64 data of an image using the console, which shouldn't be possible.

Making this game was challenging and fun and I'm definitely planning on making more multiplayer games in the future.

Get Fat Tapper

Get Fat Tapper is the first time I experimented with mobile games, I chose to work with Unity, mainly because it's a popular game engine, meaning plenty of tutorials are available online, but also because it has cross platform support so I could port my game to many platforms easily.

I thought a good way to learn unity and mobile game development in general would be to develop one of those simple yet addictive games where you click forever to gain points, but instead of just clicking I wanted the player to be able to choose which dull repetitive act they'd like to preform, so I implemented different input methods like shaking, dragging, swiping and more.

Overall the project achieved what I wanted which was to help me learn the basics of unity, and the game I made with unity afterwards was more complex and in my opinion, more engaging.


Adventure All Around

Adventure all around is the second game I made with Unity, it has the player going around the galaxy, collecting cats and moving them to their designated planets while avoiding the hostile cats. I got the idea when I played "Angry Birds Space" as a kid and thought how cool it would be if it was possible to walk on each planet and move around freely.

I like this game, I think wandering around from planet to planet is super fun and I like the little details and polish like the dust particles when the player walks or the nice depth effect in the background. On the other hand, I could have chosen a better goal for the player then moving cats around, and some of the mechanics are buggy because I lacked the necessary knowledge to implement them back then. I might revisit this project in the future.


Site Effects

Site Effects is a chrome extension I wrote in 2016 when I was 13. Around that time I heard about the filter attribute in CSS which allows you to apply basic filters to elements in a webpage like invert, grayscale, hue-rotate and more. I thought it would be interesting to make an extension which lets you apply these effects to an entire page and so began reading on how to write a chrome extension.

Back then I was quite happy with the final product, it had 9 different filter options and I also added an option to save your filter settings to local storage. When looking back at it now I see that I should have made the option to save your changes specific to each site and not global, also the code could have been written better and more efficiently.

The extension is available here if you want to try it out. Google removed the it from the Chrome Web Store, it should be back up in a couple of weeks.
